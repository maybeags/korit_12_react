## 환경 변수 이용
- CRUD를 사용하게 될 때 axios를 쓰든 아니면 fetch API를 쓰든 간에 상관없이 'http://localhost:8080/api'를 반복적으로 사용하게 될겁니다. 그런데 로컬호스트가 아니라 배포까지 하게 되면 새로운 ip 주소가 할당될거고, 도메인 구입하면 특정 문자열이 주소로 바뀔겁니다. 그러면 배포해서 ip 주소 받으면 그걸로 다 재대입하고, 또 DNS를 통해 문자열 주소 받으면 또 재대입하면서 테스트할게 아니라면 변수에다가 로컬호스트를 대입해두고 통제하는게 훨씬 효율적으로 보입니다. 그러면 URL 값이 변경되었을 때 한 곳만 수정하면 되겠네요.
비트 프로젝트의 경우 환경 변수 이름은 `VITE_`로 시작해야 합니다. 그리고, 환경변수를 저장하는 곳도 지정되어져있습니다. 루트 폴더에 `.env` 파일을 생성합니다.

- src 하위에 api 폴더 생성 -> carapi.js 라고 하는 파일 생성. API를 호출하는 함수를 모듈로 다 분리할겁니다. 현재 Carlist.tsx 내에 return 파트도 있고, caching하는 부분도 있고 함수도 있고 해서 되게 어지럽게 되어있습니다. getCars() 함수는 Carlist.tsx에서 호출만 하고, 함수 정의는 분리하겠다는 의미로 받아들이시면 되겠습니다.

- 환경변수를 불러내는 방법이 또 따로 있습니다. 비트에서 환경변수는 `import.meta.env`라는 접두사를 통해 참조가 가능합니다.

```env
VITE_API_URL=http://localhost:8080
```

```ts
import { CarResponse } from '../types'
import axios from 'axios'

export const getCars = async () : Promise<CarResponse[]> => {
    const response = await axios.get(`${import.meta.env.VITE_API_URL}/api/vehicles`);

    return response.data._embedded.cars;
  }
```
- 이상에서 함수 정의만 했는데 .env 부분의 환경변수 import 하는 예시가 적혀있습니다.
- 이하는 모듈 분리가 완료된 Carlist.tsx 부분입니다.
```tsx
import { useQuery } from "@tanstack/react-query";
import { CarResponse } from "../types";
import { getCars } from "../api/carapi";

export default function Carlist() {

  const { data, error, isSuccess } = useQuery({
    queryKey: ['cars'],
    queryFn: getCars
  });

  if(!isSuccess) {
    return <span>Loading ... ⏱️</span>
  }
  else if (error) {
    return <span>자동차 데이터를 가져오던 중 오류가 발생했습니다... 😂</span>
  }
  else {
    return(
      <table>
        <tbody>
          {
            data.map((car: CarResponse) => 
              <tr key={car._links.self.href}>
                <td>{car.brand}</td>
                <td>{car.model}</td>
                <td>{car.color}</td>
                <td>{car.registrationNumber}</td>
                <td>{car.modelYear}</td>
                <td>{car.price}</td>
              </tr>
            )
          }
        </tbody>
      </table>
    );
  }
}
```

### React Router 란 ?
- 정의 : URL을 기반으로 특정 컴포넌트를 렌더링 할지 여부를 정의할 수 있는 방식
- react-router-dom 라이브러리는 라우팅을 구현하는 데 이용되는 컴포넌트들을 제공해줍니다.

- 사용되는 Component 예시
    1. BrowserRouter : 웹 기반 애플리케이션을 이한 라우터
    2. Route : 주어진 위치가 일치하면 정의된 컴포넌트를 렌더링 // 이상에서 Route 컴포넌트의 예시를 보여줬는데, element props는 사용자가 path 프롭에 정의된 contact 엔드포인트로 이동할 때 렌더링될 컴포넌트를 정의합니다. 경로는 현재 위치를 기준으로 상대 결로를 설정합니다. 그리고 엔드포인트의 하위 URL에 또 컴포넌트들을 추가로 설정하는 것도 가능합니다. 그리고 '*'를 이용하는 것도 가능합니다(저희는 Springboot의 SecurityConfig에서 하위 엔드포인트에 `*`를 설정한 적이 있습니다). 

    예를 들어 `<Route path='/contact/* element={<Contact />} />`와 같이 사용하게 될 경우, endpoint가 `http://localhost:5173/contact/1`이거나, `http://localhost:5173/contact/kim0`이라고 설정되었을 경우에도 `<Contact />` 컴포넌트가 호출됩니다.

    또한, `<Route> </Route>` 방식으로도 쓸 수 있습니다. components 폴더 하위에 ContactSeoul 컴포넌트 생성 및 h4 태그로 서울이라고 작성해주시고 ContactBusan 컴포넌트 생성 및 h4 태그로 부산이라고 작성해주세요.

```tsx
// Contact.tsx
import { Link, Outlet } from "react-router-dom";

export default function Contact() {

  return (
    <>
      <h3>Contact Component</h3>
      <Outlet />
      <nav>
        <Link to='/contact/seoul'>서울 지점</Link> {' | '}
        <Link to='/contact/busan'>부산 지점</Link>
      </nav>
      <hr />
      
    </>
  );
}
```
`<Outlet />` : 중첩 라우트(Nested Routes)를 렌더링학기 위해 사용되는 일종의 자리 표시자(placeholder)역할을 합니다.

즉, Outlet 컴포넌트가 있는 장소에 해당 자식 컴포넌트를 불러내주세요, 라는 의미가 됩니다. 그러면 이상의 코드에서 봤던 것처럼 App.tsx의 `<Route path='contact' element={<Contact />}>`의 내부에 seoul / busan 엔드포인트를 지정했고, 거기에 ContactSeoul / ContactBunsan을 명시했습니다. 그리고 Contact 컴포넌트의 특정 위치에 Outlet 컴포넌트를 지정했기 때문에 그 위치에 ContactSeoul 이나 ContactBusan이 렌더링된다고 볼 수 있습니다.

- 이상의 라이브러리를 이용할 경우 컴포넌트들이 하나의 URL만을 가지는 것이 아니라 각각의 다른 URL을 가지도록 통제하여 컴포넌트 전체가 아니라 필요한 부분만을 렌더링하는 것이 가능합니다.

- components 폴더에 PageNotFound.tsx 생성하시고 h4로 페이지를 찾을 수 없습니다. 를 return하시오. -> App.tsx에 PageNotFound 컴포넌트를 import 하시오.


## GET 요청 Refactoring from html tags to x-data-grid
```tsx
import { useQuery } from "@tanstack/react-query";
import { getCars } from "../api/carapi";
import { DataGrid, GridColDef } from "@mui/x-data-grid";

export default function Carlist() {
  
  const columns: GridColDef[] = [
    {field: 'brand', headerName: 'Brand', width: 200,},
    {field: 'model', headerName: 'Model', width: 200,},
    {field: 'color', headerName: 'Color', width: 200,},
    {field: 'registrationNumber', headerName: 'Reg.No.', width: 200,},
    {field: 'modelYear', headerName: 'Year', width: 200,},
    {field: 'price', headerName: 'Price', width: 200,},
  ]

  const { data, error, isSuccess } = useQuery({
    queryKey: ['cars'],
    queryFn: getCars
  });

  if(!isSuccess) {
    return <span>Loading ... ⏱️</span>
  }
  else if (error) {
    return <span>자동차 데이터를 가져오던 중 오류가 발생했습니다... 😂</span>
  }
  else {
    return(
      <DataGrid 
        rows={data}
        columns={columns}
        getRowId={row => row._links.self.href}
      /> 
    );
  }
}
```
- 이상의 코드에서 AG-Grid가 아니라 X-data-grid를 도입했습니다. ColDef가 아니라 GridColDef로 정의되는 등 컴포넌트 명이 다르지만 전반적으로 유사한 코드라인을 가지고 있습니다. 그래서 특정 라이브러리에 맞는 코드를 추가 학습할 필요는 있습니다.

## DELETE 요청
- deleteCar() 메서드를 SpringBoot에서 만든다고 가정해보겠습니다.
- CarRepository라는 interface에서 해당 추상 메서드를 삭제해야할겁니다. 하지만 저희는 JpaRepository를 상속 받았었기 때문에 대놓고 쓰이지 않았었습니다. 하지만 deleteAll()과, 객체 하나만 삭제하는 메서드는 차이가 있습니다.

- DB에서의 car row를 가지고 오기 위해서 pk인 id가 있어야 한다는 점은 알지만 현재 저희가 구현한 부분에서는 id가 외부로 노출되지 않기 때문에 찾을 수 없을 수 있습니다. 이때 저희는 data._embedded_._links.self.href를 활용하여 car의 pk를 얻었습니다.

1. MUI DataGrid에 button을 렌더링하겠습니다(todolist에서 했습니다).
2. carapi.ts에 deleteCar() 함수를 정의했습니다.
3. Carlist.tsx에 deleteCar()를 import 했고, GET 요청이 아니기 때문에 UseMutation을 import하여 이하와 같이 정의했습니다.
```tsx
  const { mutate } = useMutation(deleteCar, {
    onSuccess: () => {
      //자동차 삭제 후 실행되는 로직
    },
    onError: err => {
      console.log(err);
    },
  })
```
4. GridColDefs에 mutate를 통한 함수 호출 방식을 이하와 같이 예시를 작성했습니다.
```tsx
{
      field: 'delete',
      headerName: '',
      sortable: false,
      filterable: false,
      disableColumnMenu: true,
      renderCell: (params: GridCellParams) => (
        <button onClick={() => mutate(params.row._links.self.href)}>Delete</button>
      )
    }
```
이상까지 해결을 했을 때, Delete 버튼을 눌렀을 때 Car 객체 하나가 DB에서 삭제되지만 프론트엔드 상에서는 여전히 눈에 보이고, 새로고침 했을 경우에만 X-data-grid에서 row 하나가 사라집니다.

- 이후 저희의 작업은 자동차가 삭제되면 프론트엔드를 자동으로 새로고침할 수 있도록 하는 것입니다. 그러면 useState([])에 useEffect(() => {}, [cars]); 하면 알아서 해결되는거 아닌가요 라고 할 수 있는데 그럴 수 있습니다. react-query를 사용했을 때의 자동 새로고침을 하기 위해서 useQueryClient 훅 함수를 가져와서 처리하겠습니다.

## 토스트 메시지 표시(toast message)

```tsx
import { useQuery, useMutation, useQueryClient } from "@tanstack/react-query";
import { getCars, deleteCar } from "../api/carapi";
import { DataGrid, GridColDef, GridCellParams } from "@mui/x-data-grid";
import { Snackbar } from "@mui/material";
import { useState } from "react";

export default function Carlist() {
  const [ open, setOpen ] = useState(false)
  const queryClient = useQueryClient();
  
  const columns: GridColDef[] = [
    {field: 'brand', headerName: 'Brand', width: 200,},
    {field: 'model', headerName: 'Model', width: 200,},
    {field: 'color', headerName: 'Color', width: 200,},
    {field: 'registrationNumber', headerName: 'Reg.No.', width: 200,},
    {field: 'modelYear', headerName: 'Year', width: 200,},
    {field: 'price', headerName: 'Price', width: 200,},
    {
      field: 'delete',
      headerName: '',
      sortable: false,
      filterable: false,
      disableColumnMenu: true,
      renderCell: (params: GridCellParams) => (
        <button 
          onClick={() => {
            if(confirm(`${params.row.brand}의 ${params.row.color} ${params.row.model}을(를) 삭제하시겠습니까?`)) {
              mutate(params.row._links.self.href);
            }
          }}
        >
          Delete
        </button>
      )
    }
  ]

  const { data, error, isSuccess } = useQuery({
    queryKey: ['cars'],
    queryFn: getCars
  });

  const { mutate } = useMutation(deleteCar, {
    onSuccess: () => {
      setOpen(true);
      queryClient.invalidateQueries({ queryKey: [ 'cars' ]});
    },
    onError: err => {
      console.log(err);
    },
  })

  if(!isSuccess) {
    return <span>Loading ... ⏱️</span>
  }
  else if (error) {
    return <span>자동차 데이터를 가져오던 중 오류가 발생했습니다... 😂</span>
  }
  else {
    return(
      <>
        <DataGrid 
          rows={data}
          columns={columns}
          disableRowSelectionOnClick={true}
          getRowId={row => row._links.self.href}
        /> 
        <Snackbar 
          open={open}
          autoHideDuration={2000}
          onClose={() => setOpen(false)}
          message='해당 차량 정보가 삭제되었습니다🚔'
        />
      </>
    );
  }
}
```
